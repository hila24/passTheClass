# Pass The Class 🎓

משחק אנדרואיד שנכתב ב-Kotlin — תרגיל בית 2.
את משחקת סטודנטית שמנסה לעבור את הקורס: היא נעה בין הנתיבים, מתחמקת מציוני נכשל (F) שנופלים עליה, ואוספת ציוני עובר כדי לצבור נקודות זכות.

## איך משחקים
- נעות בין הנתיבים ומתחמקות מציוני הנכשל האדומים (F).
- אוספות את ציוני העובר שמופיעים על הדרך כדי לצבור נקודות זכות.
- כל פגיעה בנכשל עולה חיים אחד. כשנגמרים החיים — נכשלת בקורס, והתוצאה נשמרת בטבלת השיאים.

## ניקוד
- התחמקות מנכשל: 5 נקודות זכות.
- איסוף עובר: 30 נקודות זכות.
- מד מרחק (Odometer) שסופר את הדרך שעברת במהלך המשחק.

## מצבי משחק (מתוך מסך התפריט)
- לחצנים — איטי
- לחצנים — מהיר
- חיישנים: הטיית המכשיר ימינה/שמאלה מזיזה אותך בין הנתיבים.
- בונוס: הטיה קדימה/אחורה משנה את מהירות המשחק.

## מסך השיאים
- מבוצע בשני פרגמנטים נפרדים:
  - טבלת עשרת השיאים הגבוהים ביותר (נשמרים מאז התקנת המשחק).
  - מפת מיקומים שמציגה היכן נרשם כל שיא, לפי הקואורדינטות.
- לחיצה על שיא בטבלה מסמנת את המיקום שלו על המפה.

## תכונות נוספות
- צליל ורטט בכל פגיעה בנכשל.
- מסלול רחב (5 נתיבים) וארוך (10 שורות).
- אייקון לאפליקציה.

## תיאור המימוש — מה עשינו

**לוח המשחק.**
הלוח בנוי כ-`GridLayout` של 5 עמודות על 10 שורות, שבו כל תא הוא `ImageView`. מצב המשחק מיוצג בשתי מטריצות בוליאניות נפרדות — אחת למיקומי הנכשלים ואחת למיקומי העוברים. בכל פריים הקוד עובר על כל התאים ומצייר אותם מחדש לפי המטריצות: שחקנית, נכשל, עובר או תא ריק. ההתקדמות נעשית על ידי הצגה/הסתרה של התאים (`visibility`), כלומר הלוח עצמו לא זז — רק התוכן שמוצג בו משתנה.

**לולאת המשחק.**
`Handler` עם `postDelayed` מריץ "טיק" קבוע. בכל טיק כל המכשולים והעוברים יורדים שורה אחת, נוצרים חדשים בראש המסלול בהסתברות אקראית, נבדקות התנגשות ואיסוף, ומתעדכן המרחק. מהירות הלולאה משתנה לפי מצב המשחק (איטי / מהיר / חיישן).

**תנועה וחיישנים.**
במצב הלחצנים, כל לחיצה מזיזה את השחקנית עמודה אחת. במצב החיישנים אנחנו מאזינים ל-`SensorManager` (חיישן התאוצה): ציר אחד שולט על תנועה ימינה/שמאלה (עם סף ו"זמן המתנה" כדי שלא תזוז יותר מדי מהר), וציר נוסף שולט על מהירות המשחק — זהו הבונוס של הטיה קדימה/אחורה.

**שמירת השיאים.**
כל תוצאה נשמרת מקומית בעזרת `SharedPreferences` בפורמט JSON (באמצעות Gson), כך שהשיאים נשמרים גם אחרי סגירת האפליקציה ומאז ההתקנה. נשמרים רק עשרת השיאים הגבוהים. יחד עם כל שיא נשמר גם המיקום שבו הושג, שנלקח דרך `FusedLocationProvider`.

**טבלה ומפה.**
מסך השיאים בנוי משני פרגמנטים: הטבלה (`RecyclerView`) והמפה (`View` מותאם אישית שמצייר ב-`Canvas`). השניים מתקשרים דרך `ViewModel` משותף עם `LiveData` — לחיצה על שורה בטבלה מעדכנת את ה-ViewModel, והמפה מאזינה לשינוי ומסמנת את הנקודה המתאימה. המפה מציגה את השיאים כנקודות לפי הקואורדינטות שלהם, מנורמלות לגודל המסך.

## טכנולוגיות
- Kotlin
- Activities + Fragments
- RecyclerView (טבלת השיאים)
- ViewModel + LiveData (תקשורת בין הטבלה למפה)
- SharedPreferences + Gson (שמירת השיאים)
- SensorManager (חיישן התאוצה)
- FusedLocationProvider (מיקום השיא)
- Canvas מותאם אישית (מפת הקואורדינטות)
